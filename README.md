## Was ist xInvoice?

xInvoice ist eine moderne Buchhaltungs-App für deutsche Kleinunternehmer und regelbesteuerte Unternehmen. Erstelle gesetzeskonforme E-Rechnungen (XRechnung & ZUGFeRD), verwalte Ausgaben, scanne Belege per KI und behalte den Überblick über deine Finanzen.

## Wie kann ich Feedback geben?

### Issues erstellen

Nutze die **Issue-Templates** um strukturiertes Feedback zu geben:

| Template | Wann verwenden? |
|----------|----------------|
| [Bug Report](https://github.com/Vibe-Technologies-EU/xinvoice-feedback/issues/new?template=bug_report.yml) | Etwas funktioniert nicht wie erwartet |
| [Feature Request](https://github.com/Vibe-Technologies-EU/xinvoice-feedback/issues/new?template=feature_request.yml) | Du wünschst dir eine neue Funktion |
| [Verbesserung](https://github.com/Vibe-Technologies-EU/xinvoice-feedback/issues/new?template=verbesserung.yml) | Eine bestehende Funktion könnte besser sein |

### Diskussionen

Für offenere Gespräche, Fragen oder Ideen die noch kein konkretes Issue sind, nutze die [Discussions](https://github.com/Vibe-Technologies-EU/xinvoice-feedback/discussions).

## Hinweise

- Dieses Repo enthält **keinen Quellcode** — es dient ausschließlich dem Feedback-Tracking
- Bitte prüfe vor dem Erstellen eines Issues, ob ein ähnliches bereits existiert
- Schreibe Issues gerne auf **Deutsch** oder Englisch
- Screenshots und Bildschirmaufnahmen sind sehr hilfreich bei Bug Reports

## Status-Labels

| Label | Bedeutung |
|-------|-----------|
| `geplant` | Feature/Fix ist eingeplant |
| `in arbeit` | Wird gerade umgesetzt |
| `erledigt` | Wurde umgesetzt und ist live |
| `wontfix` | Wird nicht umgesetzt (mit Begründung) |
| `duplikat` | Bereits als anderes Issue erfasst |
